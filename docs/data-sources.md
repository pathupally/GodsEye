# Data Sources Catalog

This catalog lists candidate open and commercial-friendly sources for multi-domain situational awareness ingestion.

| Source ID | Source name | URL | Data type | Access method | Update frequency & expected latency | License/terms & redistribution constraints | Geographic coverage & known gaps |
|---|---|---|---|---|---|---|---|
| `noaa_alerts` | NOAA / NWS Weather Alerts API | https://api.weather.gov/alerts | Weather alerts (CAP-style warnings, watches, advisories) | REST (JSON/GeoJSON) | Near real-time event-driven updates; typical publish latency 1–5 min from issuance | U.S. government work (public domain). Attribution recommended; no warranty. Follow API usage guidance and rate limits. | United States and territories; gaps outside U.S. jurisdiction. |
| `noaa_ndfd` | NOAA National Digital Forecast Database (NDFD) | https://digital.weather.gov/ | Weather forecast grids (temperature, wind, precipitation, hazards) | REST/HTTP file services (XML/GRIB2) | Hourly to sub-hourly product refresh by layer; typical operational latency 15–60 min | U.S. government data generally public domain. Redistribution allowed with source attribution and disclaimer retention. | CONUS, Alaska, Hawaii, territories, coastal waters; variable resolution by region. |
| `open_meteo` | Open-Meteo Forecast API | https://open-meteo.com/en/docs | Weather forecast/history | REST (JSON) | Model-cycle dependent (typically hourly updates); expected latency 10–90 min after model cycle | Free tier with attribution requirements; commercial terms for higher usage. Do not imply endorsement. | Global model coverage; reduced skill in sparse-observation regions and complex terrain. |
| `tomtom_traffic` | TomTom Traffic API | https://developer.tomtom.com/traffic-api | Traffic flow/incidents | REST (JSON) | Continuous updates; expected latency ~1–5 min in major metros | Commercial license/API key required; redistribution generally restricted to derived products per contract. | Broad global road network with strongest coverage in North America/Europe; rural and low-map-density gaps. |
| `osm_roads` | OpenStreetMap Planet / Extracts | https://www.openstreetmap.org/ | Roads and transport base map | PBF/CSV/GeoJSON via extracts; REST via Overpass | Minutely diff replication; full extract refresh daily/weekly depending provider; latency minutes to hours | ODbL 1.0. Share-alike required for public use of produced databases; attribution mandatory. | Global, community-maintained; completeness/accuracy uneven by country and rural areas. |
| `usace_ports` | USACE Channel Portfolio Tool (CPT) Data | https://navigation.usace.army.mil/ | Ports, waterways, channel status | REST/CSV dashboards (where exposed) and downloads | Daily to weekly depending district feed; latency hours to days | U.S. government/public domain for core data; verify district-specific terms for packaged products. | U.S. coastal/inland navigation channels; limited direct non-U.S. coverage. |
| `ais_public_feed` | AISHub / public AIS aggregators | https://www.aishub.net/ | Maritime vessel positions (AIS) | TCP/UDP stream, REST snapshots/CSV by provider | Seconds to minutes ingestion cadence; real-world latency 1–15+ min depending receiver density | Mixed terms by provider; many prohibit raw-feed redistribution without agreement. MMSI and privacy rules may apply. | Global where terrestrial/satellite receivers exist; sparse in open ocean and poorly instrumented coasts. |
| `opensky_flights` | OpenSky Network API | https://opensky-network.org/apidoc/ | Flights/aircraft states | REST (JSON) | Typical updates every few seconds to minutes; public API has throttling and delayed/limited endpoints | Non-commercial/public research oriented terms for free access; redistribution and commercial reuse constrained unless licensed. | Global ADS-B dependent coverage; poor in radar/receiver sparse regions and some state-blocked airspace. |
| `gtfs_realtime` | GTFS Realtime feeds (agency-specific) | https://gtfs.org/documentation/realtime/ | Rail/transit trip updates, vehicle positions, alerts | Protocol Buffers over HTTP (often polled as REST-like endpoint) | 5–60 sec typical producer refresh; consumer latency 10–120 sec | Agency-specific terms; many allow app use with attribution, some restrict resale/redistribution. | City/agency-specific; not universal, with frequent outages or stale feeds for smaller agencies. |
| `acled_conflicts` | ACLED API | https://acleddata.com/ | Conflicts, political violence, demonstrations | REST/CSV export | Weekly curated updates (sometimes daily for select products); latency days to weeks from events | Licensed data with registration; redistribution and commercial use restricted by ACLED terms. | Global with methodology-driven inclusion criteria; underreporting in low-media-access regions. |
| `gdal_ems` | Copernicus Emergency Management Service (GloFAS/EFAS/EMS) | https://emergency.copernicus.eu/ | Disasters (floods, fire, emergency mapping) | REST/WMS/WFS/GeoJSON downloads | Event-driven activation products; forecast layers daily/sub-daily; latency hours to day-scale | Copernicus free/open with attribution; some component layers may carry additional third-party terms. | Europe-focused for some services (e.g., EFAS) plus global layers (e.g., GloFAS); activation-based gaps. |
| `fao_giews` | FAO GIEWS and Food Price Monitoring | https://www.fao.org/giews/ | Food security/disaster impacts, commodities | CSV/REST-like downloads, reports, limited scraping | Monthly to ad-hoc bulletins; latency days to weeks | FAO content generally reusable with attribution; check per-dataset notices for commercial redistribution limits. | Global country reporting; inconsistent timeliness/detail in fragile states. |
| `worldbank_customs` | World Bank Logistics Performance / Customs indicators | https://databank.worldbank.org/ | Customs performance and border indicators | CSV/API downloads | Annual or multi-year release cadence; latency months to years (statistical publication lag) | World Bank open data terms with attribution; no implied endorsement; redistribution generally allowed. | Global country-level statistics; not suitable for operational real-time monitoring. |
| `opis_fuel` | OPIS Fuel Pricing (commercial) | https://www.opisnet.com/ | Fuel prices (retail/wholesale benchmarks) | API/CSV (contracted) | Intraday to daily depending product; latency minutes to day | Commercial subscription/license required; redistribution of raw prices typically prohibited without explicit rights. | Strong U.S. coverage, variable international products by contract; station-level completeness varies. |
